# HuggingFace TRL Training Guide

Fine-tune any open-weight model with behavioral alignment data generated by holomime.

## Prerequisites

| Requirement | Minimum | Recommended |
|-------------|---------|-------------|
| Python | 3.10+ | 3.11+ |
| GPU | — (CPU works, slowly) | NVIDIA 16GB+ VRAM (A100, RTX 4090) |
| Disk | 10 GB free | 50 GB free (model weights) |
| RAM | 16 GB | 32 GB+ |

Apple Silicon (M1/M2/M3) is supported via MPS backend — slower than CUDA but functional.

## Quick Start

```bash
# 1. Install Python training dependencies
pip install -r scripts/requirements-train.txt

# 2. Generate alignment data (if you haven't already)
holomime diagnose
holomime session          # run one or more alignment sessions
holomime export --format dpo   # or: --format alpaca

# 3. Fine-tune with HuggingFace TRL
holomime train --provider huggingface --base-model meta-llama/Llama-3.2-1B

# 4. Verify behavioral improvement
holomime eval --before .personality.json --after .personality.json
```

## Supported Base Models

Any causal language model on the HuggingFace Hub works. Recommended starting points:

| Model | Size | VRAM | Good For |
|-------|------|------|----------|
| `meta-llama/Llama-3.2-1B` | 1B | 4 GB | Quick iteration, testing |
| `meta-llama/Llama-3.2-3B` | 3B | 8 GB | Light production agents |
| `mistralai/Mistral-7B-v0.3` | 7B | 16 GB | Production agents |
| `meta-llama/Llama-3.1-8B` | 8B | 18 GB | Production agents |
| `Qwen/Qwen2.5-7B` | 7B | 16 GB | Multilingual agents |

LoRA adapters are applied automatically (r=16, alpha=32) to keep memory usage low.

## Training Methods

### SFT (Supervised Fine-Tuning)
Best when you have instruction-response pairs. Use `--format alpaca` when exporting.

```bash
holomime export --format alpaca
holomime train --provider huggingface --method sft --base-model meta-llama/Llama-3.2-3B
```

### DPO (Direct Preference Optimization)
Best when you have chosen/rejected pairs from alignment sessions. Use `--format dpo` when exporting.

```bash
holomime export --format dpo
holomime train --provider huggingface --method dpo --base-model meta-llama/Llama-3.2-3B
```

If `--method auto` (default), holomime infers the method from the export format.

## CLI Options

```
holomime train --provider huggingface [options]

Options:
  --data <path>       Path to training data (default: latest in .holomime/exports/)
  --base-model <id>   HuggingFace model ID (required)
  --method <method>   Training method: auto | sft | dpo (default: auto)
  --epochs <n>        Training epochs (default: 3)
  --suffix <name>     Model name suffix (default: agent name)
  --push              Push trained model to HuggingFace Hub
  --hub-repo <repo>   Hub repo name (e.g. yourname/agent-counselor-dpo)
  --skip-eval         Skip post-training evaluation
  --skip-deploy       Skip deploying model reference to .personality.json
  --dry-run           Show training plan without executing
```

## Pushing to HuggingFace Hub

To share your fine-tuned model:

```bash
# Set your HF token
export HUGGING_FACE_HUB_TOKEN=hf_...

# Train and push
holomime train \
  --provider huggingface \
  --base-model meta-llama/Llama-3.2-3B \
  --push \
  --hub-repo yourname/my-aligned-agent
```

The model will be saved locally AND pushed to the Hub. Without `--push`, models are saved to `.holomime/models/`.

## What Happens During Training

1. **Dependency check** — Verifies Python, PyTorch, Transformers, TRL, PEFT
2. **Data conversion** — Writes holomime export data as HF-compatible dataset
3. **Model loading** — Downloads base model + applies LoRA adapter
4. **Training** — Runs SFTTrainer or DPOTrainer with progress streaming
5. **Save** — Saves to `.holomime/models/holomime-ft-<suffix>/` (or pushes to Hub)
6. **Deploy** — Updates `.personality.json` with fine-tuned model reference
7. **Eval** — Compares base vs fine-tuned on test prompts (Behavioral Alignment Score)

## Troubleshooting

### "Missing dependency: torch"
```bash
pip install -r scripts/requirements-train.txt
```

### "CUDA out of memory"
Reduce batch size by editing `scripts/train_hf.py` (line ~155): change `per_device_train_batch_size` from 4 to 1, and increase `gradient_accumulation_steps` to 16.

### "MPS backend not available"
On Apple Silicon, ensure you have PyTorch 2.0+ built with MPS support:
```bash
pip install torch --upgrade
```

### Training is very slow on CPU
CPU training works but is 10-50x slower than GPU. For testing, use a small model like `meta-llama/Llama-3.2-1B` with `--epochs 1`. For production, use a GPU instance (Lambda, RunPod, Vast.ai).

### Model quality is poor
- Ensure you have at least 50 training examples (10 minimum, 200+ ideal)
- Try more epochs: `--epochs 5`
- DPO generally produces better behavioral alignment than SFT
- Run `holomime eval` to measure improvement quantitatively

## File Locations

| File | Description |
|------|-------------|
| `.holomime/exports/*.json` | Training data (from `holomime export`) |
| `.holomime/models/holomime-ft-*/` | Locally saved fine-tuned models |
| `.holomime/training/latest.json` | Training run metadata |
| `.personality.json` | Updated with `training.fine_tuned_model` after deploy |

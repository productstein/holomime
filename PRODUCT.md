# holomime: Open Core Model

## What's in this repo (Free, MIT Licensed)

- **Identity Stack** — 8-file behavioral identity framework
- **Therapy Engine** — reference implementation for behavioral correction
- **14 Behavioral Detectors** — rule-based drift detection (no LLM needed)
- **Robotics Adapters** — ROS2, MuJoCo, Isaac Sim, Kimodo, gRPC, MQTT
- **CLI** — 43 commands for diagnosis, therapy, training, deployment
- **Body Templates** — 8 OEM platforms (Figure 03, Unitree H1/G1, Phoenix, Ameca, Spot)
- **ISO Compliance** — 4 standards automated checking
- **NeuralActionGate** — conscience layer for learned controllers

## What's NOT here (Proprietary, holomime.com)

- **Training Infrastructure** — DPO generation at scale, model fine-tuning pipeline
- **Mira Daemon** — autonomous behavioral therapist (continuous monitoring)
- **Fleet Management** — multi-agent orchestration, supervisor-to-sub-agent spawning
- **Practice Package** — therapist model training, curriculum generation
- **Backend Services** — team management, billing, analytics
- **Production SaaS** — holomime.com dashboard, agent management, collaboration

## Why Open Core?

The Identity Stack is a **standard**, not a product. We want every robot and AI agent to have a soul.md, a conscience.exe, and a shadow.log. The more agents that adopt the standard, the more valuable the training infrastructure becomes.

**The standard is free. The training infrastructure is the business.**
